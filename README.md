# Multiagent-Based Defensive Strategy System

Real-Time Battle Field Management System. Extensible Battle Management Language. Multi-agent Deep RL for Drone Swarms. VLE is an operational framework for multi-modeling, simulation and analysis of complex dynamical systems. RCE is a distributed, workflow-based integration environment. Adversarial Machine Learning Library. RoboRTS is an open source software stack for Real-Time Strategy research. Delta3D is a simulation engine. DIS-XML, an XML Representation of Distributed Interactive Simulation Protocol Data Units. The Advanced Framework for Simulation, Integration, and Modeling (AFSIM) is a military simulation framework. The CLARITY project consists in developing and structuring an ecosystem of major actors centered on the open-source model-based systems engineering solution. The OARIS specification facilitates information exchange between the external environment and the command team, by defining a layered subsystem to the CMS interface. United Architecture Framework (UAF). Command and Control Simulation Interoperation (C2SIM). PlanMilT is a web application designed to assist in the planning of military operations at the tactical level. ODIN aims to become an alternative to industry-led commercial Command and Control Information Systems. C Model Developer CMD is an open-source C source code based environment for building simulations of systems described by time-based differential equations. Boomslang is a C2 simulator that provides an open-source tool to simulate new C2 doctrines and exercises. PythonPDEVS is a Python implementation of the Parallel Discrete Event System Specification (PDEVS) formalism. ModelicaDEVS is a free library for discrete-event modeling. Protocol for composing Distributed DEVS Models Across different Simulation Frameworks. The ARL Hierarchical MultiScale Framework (ARL-HMS) is a software library for development of multiscale models on heterogeneous high-performance computing systems. OpenMAS streamlines asynchronous Multi-Agent System (MAS)

The directory structure includes a specific folder for each phase of development and engineering programs:

- __MBSE__: The root folder Arcadia (ARChitecture Analysis and Design Integrated Approach) is a method of tools dedicated to systems engineering and architecture, supported by the Capella modeling tool.    
  - __CAD__: For computer-aided design.
  - __CAM__: Dedicated to the part manufacturing and assembly process.
  - __CAS__: For simulation of end-to-end functionality and performance analysis.
